# UX/UI 가이드라인

> 트레이딩 대시보드 특화. UX 검증 시 `ux-reviewer` 에이전트 활용.

## 1. 상태 표시 필수

모든 데이터 컴포넌트는 4가지 상태를 처리:

| 상태 | 패턴 | 필수 |
|------|------|------|
| Loading | `<Loading />` 또는 Skeleton UI | ✅ |
| Error | `<ErrorBanner message={...} onRetry={...} />` | ✅ |
| Empty | 안내 메시지 + 행동 유도 (CTA) | ✅ |
| Success | 데이터 렌더링 | ✅ |

## 2. 접근성 (a11y)

- 아이콘 전용 버튼에 `aria-label` 필수
- 색상만으로 정보 전달 금지 → 수익: 초록+▲, 손실: 빨강+▼
- 키보드 네비게이션 지원 (`tabIndex`, `onKeyDown`)
- 폼 입력에 `<label>` 연결 (htmlFor)
- 최소 명암비 4.5:1

## 3. 데이터 표시 포맷

| 데이터 | 포맷 | 예시 |
|--------|------|------|
| 가격 (KRW) | 천 단위 콤마 | 52,300원 |
| 가격 (USD) | $, 소수점 2자리 | $1,234.56 |
| 수익률 | %, 소수점 2자리, 색상+화살표 | +12.34% ▲ |
| 날짜 | YYYY-MM-DD HH:mm | 2026-02-10 14:30 |
| 큰 숫자 | 축약 표시 | 1.2억, $1.2M |

## 4. 인터랙션 패턴

- 폼 제출 시 이중 클릭 방지 (`disabled` + loading)
- 삭제/위험 작업 시 확인 모달 필수
- 실시간 데이터는 깜빡임 없이 부드럽게 업데이트
- 토스트: 성공(3초 자동 닫힘), 에러(수동 닫힘)

## 5. 레이아웃

- 데스크탑 우선 (min-width: 1280px)
- 사이드바: 고정 너비 (`w-64`)
- 메인: 반응형 (`flex-1`)
- 모바일: 핵심 데이터만 표시, 사이드바 → 햄버거

## 6. Tailwind 일관성

| 속성 | 기준값 |
|------|--------|
| 간격 | `p-4`, `gap-4` (4 단위) |
| 모서리 | 카드: `rounded-lg`, 버튼/입력: `rounded-md` |
| 그림자 | 카드: `shadow-sm`, 모달: `shadow-md` |
| 다크 모드 | `dark:` 접두사 지원 |
